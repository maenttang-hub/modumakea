# ModuMake Design Alignment

이 문서는 현재 제품 방향과 예전 구현이 충돌하던 지점을 정리하고, 어떤 축부터 최신화했는지 기록합니다.

## 현재 기준

ModuMake의 imported schematic 화면은 더 이상 회로를 새로 그리는 편집기가 아니라, 기존 KiCad 회로도를 불러와 검토하고 저장하고 의견을 남기는 리뷰 화면으로 간주합니다.

핵심 흐름은 아래 세 가지입니다.

1. `.kicad_sch`를 바로 불러온다.
2. 연결 상태와 검증 결과를 먼저 본다.
3. 필요한 위치에 댓글과 피드백을 남긴다.

## 예전 구현과 안 맞던 부분

### 1. 설계 도구처럼 보이는 메뉴와 문구

imported schematic 화면인데도 일부 메뉴와 우클릭 동작은 예전 캔버스 편집기 관성을 그대로 갖고 있었습니다.

대표적인 예:

- `작업 메뉴`
- `부품 라이브러리`
- Undo / Redo 강조
- 저장 모드 토글
- 캔버스 작업 중심 우클릭 메뉴

이 요소들은 "리뷰 화면"보다 "설계 화면"처럼 느껴지게 만들었습니다.

### 2. imported review와 새 설계용 사이드바의 혼선

왼쪽 패널 안에는 이미 imported schematic 전용 리뷰 요약이 들어와 있었지만, 일부 문구는 여전히 "부품 카탈로그" 관점에 머물러 있었습니다.

이 때문에 사용자는:

- 여기서 새 부품을 고르는 건지
- 아니면 기존 회로를 검토하는 건지

를 한 번 더 해석해야 했습니다.

### 3. 리뷰 모드와 편집 모드의 행동 차이 부족

imported schematic에서는 댓글, 검증, 화면 맞춤 같은 행동이 중심이어야 하는데, 우클릭 메뉴나 상단 메뉴 일부는 예전 편집 액션을 계속 보여주고 있었습니다.

## 이번에 맞춘 부분

### imported review 문법 정리

- imported schematic일 때 우클릭 메뉴 제목을 `도면 리뷰` 기준으로 바꿨습니다.
- imported schematic의 캔버스/보드 우클릭 메뉴에서는 댓글과 검증 중심 액션만 남기고, 예전 편집기 성격의 액션 노출을 줄였습니다.
- imported schematic의 사이드바 우클릭 메뉴도 `검증 보기`, `주석 열기`, `화면 맞춤` 위주로 단순화했습니다.

### 상단 작업 메뉴 단순화

- imported schematic일 때는 저장 방식 토글을 숨겼습니다.
- imported schematic일 때는 Undo / Redo 버튼 묶음을 숨기고, 대신 이 화면이 리뷰/저장/댓글 흐름 중심이라는 안내로 바꿨습니다.

### 왼쪽 패널 어조 정리

- `부품 카탈로그는 숨김` 문구를 `새 설계 기능은 숨김`으로 바꿨습니다.
- 설명도 "새 부품을 고르는 화면"이 아니라 "기존 회로를 검토하고 의견을 남기는 화면"이라는 쪽으로 정리했습니다.

## 아직 남은 축

이번 정리는 화면의 역할과 문법을 맞춘 단계입니다. 아직 아래 축은 계속 진행해야 합니다.

1. 저장 후 재로드에서도 wire / junction / label 좌표가 절대 안 흔들리게 유지
2. primitive가 있는 심볼은 fallback이 거의 끼지 않도록 줄이기
3. MCU / connector / power symbol의 텍스트 정렬을 KiCad에 더 가깝게 맞추기
4. validation 패널 내부도 리뷰 중심 구조로 더 얇게 분리하기

## 원칙

앞으로 imported schematic 경로에서 새 기능을 추가할 때는 아래 원칙을 우선합니다.

- 렌더 축과 검증 축을 섞지 않는다.
- 리뷰 화면에서는 설계 액션보다 검토 액션을 먼저 보여준다.
- legacy fallback은 줄이고, 원본 KiCad 데이터 우선 비율을 높인다.
- 저장 후 다시 열어도 같은 좌표, 같은 연결, 같은 리뷰 상태를 보장한다.
